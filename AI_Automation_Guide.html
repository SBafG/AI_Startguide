Automation & triggers

Hur vet systemet att en ny CSV finns?

Tre sätt att trigga ditt script automatiskt — från enklast till mest robust

Det är en bra fråga. Du har ett portföljsystem som exporterar en CSV-fil en gång i månaden. Du har ett script som läser filen, skickar till Claude och publicerar rapporten. Men vad startar scriptet?

Det finns tre huvudsakliga sätt — välj det som passar din miljö:

De tre vanligaste triggermetoderna

⏰
Schemalagd tid
Kör varje dag kl 08:00, varje måndag, första dagen i månaden osv.
✓ Enklast att sätta upp
Windows Task Scheduler, cron (Mac/Linux) eller Power Automate
👁️
Filbevakning
Scriptet vaktar en mapp — reagerar direkt när en ny CSV dyker upp.
Rekommenderas
✓ Reagerar omedelbart
Node.js fs.watch() eller chokidar-paketet
🔔
Webhook / signal
Källsystemet skickar en HTTP-signal till ditt script när export är klar.
✓ Mest pålitligt
Kräver att källsystemet stödjer webhooks
Enklaste starten: Schemalagd tid med Windows Task Scheduler — du pekar ut scriptet och säger "kör varje måndag 08:00". Inget mer. Om CSV-filen inte förändrats sedan sist kan scriptet hoppa över körningen.

Alternativ 1 Filbevakning med Node.js

Det här scriptet vaktar mappen C:\Rapporter\export\. Så fort en ny .csv-fil skapas eller ändras startar det automatiskt hela kedjan — läser filen, skickar till Claude, publicerar.

watcher.js Node.js 
const fs    = require('fs');
const path  = require('path');
const { generateReport } = require('./report');

const WATCH_FOLDER = 'C:/Rapporter/export';

console.log(`Bevakar mappen: ${WATCH_FOLDER}`);

fs.watch(WATCH_FOLDER, async (eventType, filename) => {
  // Reagera bara på CSV-filer
  if (!filename?.endsWith('.csv')) return;

  const filepath = path.join(WATCH_FOLDER, filename);

  // Vänta 1 sek så filen hinner skrivas klart
  await new Promise(r => setTimeout(r, 1000));

  console.log(`Ny fil hittad: ${filename} — startar rapport...`);

  try {
    await generateReport(filepath);
    console.log('✅ Rapport klar och publicerad.');
  } catch (err) {
    console.error('❌ Fel:', err.message);
  }
});

Scriptet körs en gång (node watcher.js) och sedan lyssnar det tills du stänger det — eller så lägger du det som en bakgrundstjänst med PM2.

Alternativ 2 Schemalagd körning

Enklare variant — scriptet körs vid en fast tid, kontrollerar om CSV-filen är ny sedan sist och agerar då.

report.js Node.js 
const fs          = require('fs');
const Anthropic    = require('@anthropic-ai/sdk');

const CSV_PATH     = 'C:/Rapporter/export/portfolio.csv';
const LAST_RUN    = 'C:/Rapporter/.last_run';
const client      = new Anthropic({ apiKey: process.env.ANTHROPIC_API_KEY });

async function run() {

  // 1. Kolla om filen är nyare än senaste körning
  const fileModified = fs.statSync(CSV_PATH).mtime;
  const lastRun      = fs.existsSync(LAST_RUN)
    ? new Date(fs.readFileSync(LAST_RUN, 'utf8'))
    : new Date(0);

  if (fileModified <= lastRun) {
    console.log('Ingen ny data sedan sist — hoppar över.');
    return;
  }

  // 2. Läs CSV-data
  const csvData = fs.readFileSync(CSV_PATH, 'utf8');

  // 3. Skicka till Claude — be om månadskommentar
  const response = await client.messages.create({
    model:      'claude-haiku-4-5-20251001',
    max_tokens: 600,
    messages: [{
      role:    'user',
      content: `Du är en portföljanalytiker. Här är månadens portföljdata som CSV:

${csvData}

Skriv en professionell månadskommentar (max 150 ord) som:
- Sammanfattar avkastningen vs benchmark
- Lyfter fram de viktigaste bidragsgivarna (pos/neg)
- Avslutar med en neutral framåtblick
Svara på svenska.`
    }]
  });

  const comment = response.content[0].text;

  // 4. Skriv rapport-HTML till portal-mappen
  const html = buildReportHTML(csvData, comment);
  fs.writeFileSync('C:/Portal/monthly_report.html', html);

  // 5. Spara tidsstämpel
  fs.writeFileSync(LAST_RUN, new Date().toISOString());

  console.log('✅ Rapport genererad och sparad.');
}

run();

Praktisk setup — kom igång

Det du behöver installera:

  • Node.js — gratis, ladda ner från nodejs.org
  • Anthropic SDK — npm install @anthropic-ai/sdk
  • PM2 (om du vill att watchers körs som bakgrundstjänst) — npm install -g pm2
Terminal Bash 
# Starta watcher som bakgrundstjänst
pm2 start watcher.js --name "portfolio-watcher"

# Starta automatiskt vid omstart av datorn
pm2 startup
pm2 save

# Se loggar
pm2 logs portfolio-watcher

Med PM2 körs bevakaren i bakgrunden och startar om automatiskt om datorn startas om — du behöver inte tänka på det.

En sak till — API-nyckeln

Scriptet behöver din Anthropic API-nyckel för att anropa Claude. Lägg den som en miljövariabel — inte direkt i koden.

.env Config 
ANTHROPIC_API_KEY=sk-ant-api03-din-nyckel-här

Hämta nyckeln på console.anthropic.com → API Keys. Kostnaden för en månadsrapport med Claude Haiku är under 1 kr.
